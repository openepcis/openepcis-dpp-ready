# GEFEG BatteryPass-Ready — live-derived validation schemas

These four JSON schemas reproduce what the **live** GEFEG `ValidateJSON` server
(https://batterypass-ready.gefeg.com/, retrieved 2026-06-17) actually enforces —
which is **not** what GEFEG publishes for download under
[`../../docs/reference/gefeg-batterypass-ready/`](../../docs/reference/gefeg-batterypass-ready/).

| File | Category | Live API tag | Root key |
|------|----------|--------------|----------|
| `EV.schema.json` | Electric Vehicle | `EV_BatteryPass` | `EV` |
| `LMT.schema.json` | Light Means of Transport | `LMT_BatteryPass` | `LMT` |
| `OtherIndustrial2kWh.schema.json` | Other Industrial > 2 kWh | `Other_Industrial_BatteryPass` | `OtherIndustrial2kWh` |
| `StationaryIndustrial2kWh.schema.json` | Stationary Industrial > 2 kWh | `Stationary_Industrial_BatteryPass` | `StationaryIndustrial2kWh` |

## How the published static files differ from the live server

- **No `required`.** The static files declare none; the live server requires
  ~77–89 properties per category.
- **Wrong Stationary root key.** Static uses `Industrial2kWh`; the live server
  requires `StationaryIndustrial2kWh`.
- **Per-category required-sets differ** (the static files are byte-identical
  except the root key):
  - EV uniquely requires `StateOfCertifiedEnergySOCE`.
  - LMT and Stationary additionally require `DateOfPuttingTheBatteryIntoService`
    plus the extended performance set (`RemainingCapacity`,
    `RemainingPowerCapability`, `RemainingRoundTripEnergyEfficiency`,
    `EvolutionOfSelf-dischargeRates`, `EnergyThroughput`, `CapacityThroughput`,
    `TimeSpent…ExtremeTemperatures…` ×4, `NumberOfDeepDischargeEvents`).
  - Other Industrial is the leanest.
- **Key-name / shape corrections**: enum objects use `<name>Value`
  (`dppStatusValue`="Active", `batteryCategoryValue`∈{"electric vehicle battery",
  "LMT battery","industrial battery"}, `batteryStatusValues`="original");
  hyphenated names (`ExpectedLifetime-NumberOfChargeOrDischargeCycles`); power at
  80/20 SoC; and the integer vs decimal ampere-hour variants disagree on unit-key
  casing (`ampereHourMiliamperehour` vs `amperehourMiliamperehour`).

## Provenance / regeneration

Generated by `scripts/build-gefeg-live-schema.ts`: required-sets come from
probing the live API with empty groups (`scripts/probe-gefeg-required.ts`);
value-object/enum shapes come from the verified-valid fixtures in
`../../examples/batterypass-ready/<category>.gefeg.json`. Regenerate with
`pnpm run build:gefeg-live-schema`. The authoritative check is still the live
API (`pnpm run validate:batterypass-live`); the offline `pnpm test` validates the
fixtures against these derived schemas.

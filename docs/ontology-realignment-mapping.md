## Methodology

Each extension term is matched against the cached GS1 Web Vocabulary (`gs1:`, ~600 terms) 
and schema.org (`schema:`, ~2400 terms) using local-name and label tokens. 
Per governance precedence, GS1 ranks before schema.org when both match. 
Schema.org meta-types (Number, Date, Class, DataType, etc.) are blocked as match targets. 
Property/Class kind mismatches are demoted to LOW confidence.

**Confidence is heuristic.** HIGH means exact name/label match, has-prefix match, or strong token overlap (Dice ≥ 0.75). 
It is *not* a guarantee — every REPLACE_WITH_* row needs domain-expert review before TTL edits.

## Verdict legend

| Verdict | Action |
|---|---|
| `REPLACE_WITH_GS1` | Delete from TTL. Replace usages with GS1 term. |
| `REPLACE_WITH_SCHEMA` | Delete from TTL. Replace usages with schema.org term. |
| `KEEP_LINK_GS1` | Keep extension term, add owl:equivalentProperty/Class or rdfs:subPropertyOf to GS1. |
| `KEEP_LINK_SCHEMA` | Same against schema.org. |
| `KEEP_NO_LINK` | No external equivalent found; review whether to keep, rename, or research further. |

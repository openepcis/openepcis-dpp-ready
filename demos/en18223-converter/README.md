# EN 18223 converter demo

A simple, self-contained web page that demonstrates the **EPCIS4DPP**
derivation: paste good **GS1 Web Vocabulary + GS1 Digital Link JSON-LD**
(the EN 18223 *compressed* serialization) and see the derived EN 18223
*expanded* `DigitalProductPassport` (Annex A) on the right. Each data point
becomes a `DataElement` whose `dictionaryReference` resolves into the
ref.openepcis.io data dictionary; `valueDataType` comes from the ontology and
`granularity` is derived from the Digital Link Application Identifiers
(`01` → model, `01`+`10` → batch, `01`+`21` → item).

Everything runs **client-side**. The page reuses the same derivation core as
the CLI (`scripts/en18223/derive-core.ts`); only the document loader and the
range index differ (bundled in for the browser).

## Build and run

```bash
pnpm run demo:en18223:build     # build range-index.json + bundle app.js
# then serve the folder with any static server, e.g.:
npx serve demos/en18223-converter
# or, build + serve with esbuild's dev server:
pnpm run demo:en18223
```

Open the served `index.html`. `app.js` is a committed prebuilt bundle, so the
page also works from any static host without rebuilding.

## Hosting on GitHub Pages

The page is fully static (relative asset paths, no runtime network calls for the
bundled samples), so it deploys to GitHub Pages as-is. The workflow
`.github/workflows/pages.yml` builds the bundle and publishes this folder on
every push to `main`. One-time setup: repo **Settings → Pages → Source: GitHub
Actions**. It then serves at `https://<org>.github.io/<repo>/`.

## Files

- `index.html`, `styles.css`: the page.
- `app.ts`: browser entry; bundled to `app.js` by esbuild
  (`--loader:.jsonld=json`, `--platform=browser`).
- `range-index.json`: generated by `scripts/build-en18223-rangeindex.ts`
  (property IRI → `rdfs:range`).
- `app.js`: prebuilt bundle (includes `jsonld`, the dpp-core context, the
  range index, and the sample inputs).

The derivation logic lives in `scripts/en18223/derive-core.ts` (shared with the
CLI `scripts/derive-en18223.ts`). Not official GS1 or CEN/CENELEC guidance.
